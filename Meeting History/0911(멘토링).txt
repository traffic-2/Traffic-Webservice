1. 지역을 어떻게 선정해서, 어떤 데이터를 통해 어떤 y값을 얻을 것인지 명확하게 하기

2. 분석범위가 너무 크다
=> 핵심지역을 하나 선택해서 그 지역에 대해 자세하게 (지역을 좁혀서 사고건수, 유형도)
ex) 서울시 내에서도 사고가 가장 많이 일어난 지역 ?개 선정해서 분석 진행
ex) 지역구 내에서도 특정 지점만 선정해서 진행
ex) 자신있는 핫스팟 몇군데만 선정해서 분석 진행
=> 지역구를 선정한 이유는? 선택한 이유를 충분히 설명할 것
=> 범위가 좁혀지는 곳 대해서만 웹서비스 구현 부분
=> 교통량 측정되는 위치 위주로 진행 / 지점명 위도, 경도 좌표에서 교통량 알수 있는지 먼저 확인 필요

3. 최종 목적
- 사고가 많이 나는 지점을 찾아서 그 지점에 대해 잘 알려주는 것이 목적 (기존 지점에 대한 새로운 정보를 주는 것이 목적)

4. 웹사이트 우선순위
- 서울시 내(전체)의 교통사고 데이터 분석 결과 시각화해서 웹사이트에 개시
- 서울시 내에서 지역구를 좁혀서 머신러닝 적용
- 현재 위치 기준으로 주변 교통사고 위치, 위험도 나타내는 서비스
=> 사고유형에 따라 색깔을 나눠 알 수 있도록 보완해주면 더 좋을것
=> 위험지수 선정기준 어떻게 나눌 것인지 고려
- 경로 (최단거리 & 우리가 만든 안전거리 로직)
A에서 B로 가는데 보행, 차량/성별/연령에 따라 사고가 발생하지 않는 안전한 길을 추천하는 시스템에 활용할 수 있다는 확장성 측면으로 제시(개인 맞춤형)

6. 차별점
- 사고다발지역이라는 정보랑 우리가 분석해서 나타내고자 하는 아웃풋이 얼마나 차이가 있는지?
- 예측하고자하는 위험지수, 안전경로 부분이 기존에 알고자하는 부분에서 얼마나 업그레이드, 차별점이 뭔지?
- 실시간 교통량, 날씨, 시간에 바탕으로 해당 사고지점의 위험도를 표시 => 사고건수 위험도랑 비교한거랑 많이 다름?
- 기존은 사고다발지지역의 유무를 나타냈지만 => 이걸 세분화해서 80%,70%로 나눔에 따라 업그레이드 된 부분이 이거다라고 말할게 있어야함
- 어떤 대상을 위해 만들 수 있는 분석을 목적으로 하는지?
ex) 운전자 맞춤형 사고다발지역
- 우리의 안전경로 안내가 기존과의 차이가 무엇인지? (기존에서 사고다발지를 알려주는 기능이 있음)
- 경로1,2,3을 선택해라 => 사용자가 선택하게되면, 보통은 최적지역을 많이 선택해서 볼수있을것
