[ 1차 피드백 ]

1. 질병예측 주제
- 의료 데이터 개방이 잘 안돼서 어렵고 수집하기 힘듦
- 건강보험공단 자료 => 단순 건강검진 자료 (실시간 자료x)
- 건강보험공단자료 이외에 건강영양조사 참고해도 좋음
- 흡연, 음주 관련해서 이미 많이 논문들이 있음 => 차별화된 부분이 있어야함
- 당뇨 수치가 높게 나온게 일시적인지, 유지적인지에 따라 진단이 달라짐 (의료진 의견이 중요)
- ex 고혈압, 당뇨 진단 요건 정의로 자체적으로 진단 내리고(정의를 내려버리기) 1년 있다, 2년있다 걸릴 가능성을 예측

- 멘토님 추천 주제
- 정상 수치를 넘어가는 사람에 대한 탐지 (이상징후 탐지)
- 각 질병에 대한 정상범위에서 벗어난 사람들을 체크해주는 것은 어떨지
- 이후에 스마트워치에 접목시키면
- 패턴분류
- 마찬가지로 차별성 필요
- 직업별 혈압수치 / 지역별 , 연령별 혈압수치
- 건강영양조사 10년치를 가져와서 정상치를 조사해서 알아내기

- 해외데이터 사용해도 괜찮음
- 결과를 제시하기보다도 과정을 제시
- 비록 데이터 얻기 어려워서 알려져있는 해외데이터를 썼지만, 국내에서 데이터를 자유롭게 쓸 수 있다면 전국민의 이상치를 얻어내는 데에 효율적일 것이다


2. 균열탐지 주제
- 건물을 어떻게 다 스캔할 것이냐라는 문제가 나올 수 있음 (현실적으로 가능할지)
=> 꼭 드론이 아니더라도 정책적으로 앞으로 몇층이상 되는 건물에는 해당 소프트웨어(시스템)을 설치할 것을 제안할 수도 있음
=> 드론도 하나의 제시책일뿐

- 건축종류에 따라, 균열여부에 따라 위험한 균열인지 아닌지를 나눠질수 있음
- 특정 부분의 특정 재료를 사용했을 때 이런 결과가 나온다 이러면 좋지만 조사할 부분이 너무 많을것
- 자재 종류별로 다른건 어떡할지 ? => 우리 프로젝트는 한정적이지만 추후 자재별에 따라 기준을 다르게 세우고 적용하면 될 것

- 재난청 : 균열이나 담이 몇도 넘어갔느냐에 따라 라벨링이 되어있음
- 재난 상황을 미리 예측할 수 있는 방향
- 균열이라 하더라도 전체 균열 중에서 일부분을 떼서 한달이라는 시간안에 할 수 있는 데이터를 만들어놓고 하면 좋을 것
- 실생활에 어떻게 적용될지를 고려하기
- 균열탐지도 실생활에 하려면 사진도 찍어보고 동영상도 찍어보고 우리가 만든 모델링이 활용될 가능성이 있음을 보여주기

- 타겟을 너무 크게 잡기보다도 할 수 있는 부분을 찾아내서 하기
- 굳이 드론이라고 콕 찝어서 주제를 정할 필요는 없음


3. 오픈마켓 주제
- 독창성 => 소비자를 위한 맞춤추천일지, 판매자를 위한 맞춤추천일지 정하면 좋음 (누가 타겟이냐가 중요)
- 구매율 대신 리뷰수 파악 : 자료가 편향될 가능성이 있음
- 네이버 분석과 차별된 부분에 좀 더 포커싱해서 부각시키기

- 리뷰건수도 그렇고 점수를 주는 건 현재의 상황
- 시기별로 튀어오르는 건지 갑자기 오르는 건지 꾸준히 팔리는건지를 알 수 없음
- 단순 현재의 상황만 갖고 판단하기에는 애매 => 일정 기간동안 데이터를 모으고 그걸 바탕으로 예측 진행


4. 기타
- 기술스택 관련 질문 : 가고자 하는 그룹들이 무슨 기술스택을 갖고있는지 살펴보기
- 어떤 알고리즘을 사용하는게 좋을지 : 최대한 마무리될때까지 잘나오는 모델링을 찾아서 비교
=> 데드라인 정해놓고 최대한 많이 돌려보기
=> 교육 때 진행했던 모델링 다 돌려보기
=> 데이터는 시간이 많이 투자하고 많이할수록 더 좋은 결과를 얻을 수 있다

- 포장하는 것도 중요 (해석을 얼마나 잘하냐가 중요)
- 분석을 할때 결과가 잘 안나와도 괜찮음, 어떻게 해석하느냐에 따라 충분히 커버할 수 있음
- 앞으로의 향후계획, 아이디어가 얼마나 창의적인지
- 세가지 주제에서 모두 지금까지와의 차별성을 생각하기

[ 2차 피드백 ]

1. 질병 예측 모델
- 데이터셋이나 방향자체를 조금 바꿔야될 것이라는 생각
- 질병쪽은 사실 의사도 오진을 하는 분야
- 내가 해당 분야의 도메인의 전문가가 되어야 예측이 가능한 것
- 의료 데이터는 오픈이 잘 되지 않음
- 너무 어려운 분야라고 생각
=> 데이터를 어떻게 확보할 수 있느냐가 문제


2.
- 드론이 차별성을 가진 주제성을 가질 수 있을것인가?
- 이미지 디텍딩을 통한 불량률 탐지 모델 => 굉장히 노말한 주제로 바뀌는 느낌
- 차별성이 드론말고 다른게 있을까?
=> 노말한 주제를 어떻게 특성을 더한 주제로 만들 수 있느냐가 문제


3.
- 물품의 구매예측을 판매자의 등급으로만 판단할 수 있을까?
- 구매예측을 어떻게 할 것인지? => 조회수로
- 처방은 어떻게 할 것인지? => 상세페이지에 대한 제품 설명 부분에서 이미지수, 글자수, 동영상수, 리뷰상태 부분 처방
- 처방예시 : 상세페이지 정보량을 늘려야된다... 연관채널 잘 꾸미거나... 사진을 잘 찍는법... 
- 처방이 명확하지 않다
- 처방을 하기 위한 가설이 필요할 것
- 가설이 명확하지 않으면 처방이 명확하지 않을 것이다
=> 처방이라는 고차원적인 분석을 위해 어떻게 시나리오와 가설을 잡고 다양한 방면에서 처방을 내릴 수 있느냐가 문제


4. 최종 주제 선택
두번째 주제가 가장 수월할 것 (건물 균열 탐지 프로젝트)


5. 기타
- 탐지와 예측은 다름
=> 탐지는 균열인지 아닌지를 찾는 것
=> 예측은 앞으로 균열된 이미지와 부가적인 데이터가 필요할 것 (평상시 받는 무게나 중량, 이용하는 사람의 규모 등)
=> ex) 화재탐지 : 화재판단 기준이 여러가지
=> 균열탐지에서 차별화된 점은 도메인적인 지식이 필요

- 포커싱을 바꿔서 새로운 주제를 탐색해도 좋음
=> 기업들이 ESG쪽으로 방향을 틀고 있는 추세
=> 환경과 관련해서 (폭염, 장마, 홍수같은 자연재해) / 폭염예측, 장마예측, 홍수예측
=> 홍수나 자연재해로 인해 피해가 많은 지역을 분류하는 것도 괜찮음
